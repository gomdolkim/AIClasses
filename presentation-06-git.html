설정

최초 1회 설정 (내 정보 등록)

# 이름 설정 (GitHub 이름과 같게)
$ git config --global user.name "홍길동"

# 이메일 설정 (GitHub 이메일과 같게)
$ git config --global user.email "hong@email.com"

💡 이 설정은 딱 한 번만 하면 됩니다. 모든 커밋에 이 정보가 기록돼요.

명령어

① git init — 저장소 만들기

프로젝트 폴더에서 Git을 처음 시작하는 명령어

• 프로젝트당 1번만 실행

• 숨겨진 .git 폴더가 생성됨

• 이 순간부터 변경 추적 시작!

# 프로젝트 폴더로 이동
$ cd my-website

# Git 시작!
$ git init
Initialized empty Git
repository
12 / 30
명령어

② git status — 현재 상태 확인

어떤 파일이 수정/추가됐는지 확인하는 명령어

• 빨간색 = 아직 add 안 한 파일

• 초록색 = add 완료, 커밋 대기 중

• 수시로 확인하는 습관!

$ git status

Untracked files:
  index.html
  style.css

💡 길을 잃었을 때의 나침반! 헷갈리면 git status부터 실행하세요

13 / 30
명령어

③ git add — 스테이징에 올리기

# 특정 파일만 추가
$ git add index.html

# 모든 변경된 파일 추가 (가장 많이 사용!)
$ git add .

git add index.html

이 파일만 찍을 준비

git add .

전부 다 찍을 준비! (추천)

14 / 30
명령어

④ git commit — 스냅샷 저장!

# 메시지와 함께 커밋 (-m = message)
$ git commit -m "첫 번째 페이지 완성"

✍️ 좋은 커밋 메시지 작성법

❌ 나쁜 예

"수정함"

"asdf"

"ㅋㅋ"

✅ 좋은 예

"메인 페이지 헤더 추가"

"로그인 버그 수정"

"배경색을 파란색으로 변경"

15 / 30
명령어

⑤ git log — 커밋 이력 보기

$ git log --oneline

a1b2c3d 배경색을 파란색으로 변경
e4f5g6h 메인 페이지 헤더 추가
i7j8k9l 첫 번째 페이지 완성

💡 a1b2c3d 같은 코드 = 커밋 ID. 이걸로 과거 버전을 찾아갈 수 있어요!

나가려면 q 키를 누르세요

실습 타임!

실습: 기본 명령어 따라하기

# 1. 폴더 만들고 이동
$ mkdir git-practice && cd git-practice

# 2. Git 시작
$ git init

# 3. 파일 만들기
$ echo "Hello Git!" > index.html

# 4. 상태 확인 → 추가 → 커밋
$ git status
$ git add .
$ git commit -m "첫 커밋"
$ git log

⏱️ 5분간 함께 진행합니다!

GitHub 연동

git remote — GitHub 주소 연결

# GitHub 레포 주소를 "origin"이라는 이름으로 등록
$ git remote add origin https://github.com/내아이디/my-website.git

💡 "origin" = GitHub 주소의 별명. 관례적으로 항상 origin이라고 씁니다.

GitHub 연동

git push — 코드 올리기

내 컴퓨터의 커밋을
GitHub에 업로드

💻 내 PC
→
🐙 GitHub
# 처음 push할 때
$ git push -u origin main

# 그 다음부터는
$ git push
20 / 30
GitHub 연동

git pull — 코드 가져오기

GitHub에 있는 최신 코드를
내 컴퓨터로 다운로드

🐙 GitHub
→
💻 내 PC
# 최신 변경사항 가져오기
$ git pull

💡 push = 밀어 올리기 / pull = 당겨 받기. 반대 방향!

21 / 30
GitHub 연동

git clone — 프로젝트 통째로 복사

# GitHub의 프로젝트를 내 컴퓨터로 복사
$ git clone https://github.com/아이디/프로젝트.git

✅ 폴더 생성 + 코드 다운로드 + Git 설정 — 한 번에 완료!

💡 새 컴퓨터에서 작업하거나, 다른 사람의 프로젝트를 가져올 때 사용합니다

22 / 30
워크플로우

매일 쓰는 Git 루틴

# 1. 코드 수정 후 상태 확인
$ git status

# 2. 변경 파일 모두 스테이징
$ git add .

# 3. 커밋 (스냅샷 저장)
$ git commit -m "오늘 작업 내용"

# 4. GitHub에 올리기
$ git push

🔄 status → add → commit → push — 이 4단계만 기억하세요!

코드의 평행 우주

24 / 30
브랜치
브랜치 (Branch)
"가지"라는 뜻
원본 코드를 건드리지 않고 새로운 실험을 할 수 있는 복사본
main
new-feature

💡 비유: 게임 세이브 슬롯 — 메인 세이브는 안전하게 두고 새 슬롯에서 실험!

25 / 30
브랜치

브랜치 핵심 명령어

# 새 브랜치 만들고 이동
$ git checkout -b new-feature

# 브랜치 목록 보기 (* = 현재 위치)
$ git branch
  main
* new-feature

# main 브랜치로 돌아가기
$ git checkout main
26 / 30
브랜치

git merge — 합치기

실험이 성공하면 main에 합칩니다

# 1. main으로 이동
$ git checkout main

# 2. new-feature를 main에 합치기
$ git merge new-feature

# 3. 합친 결과를 GitHub에 올리기
$ git push

💡 실험 실패? 브랜치만 삭제하면 됩니다. main은 안전! → git branch -d new-feature

Git 명령어 한눈에 보기

기본

git pull — GitHub에서 받기

git clone URL — 프로젝트 복사

git checkout -b 이름 — 브랜치 생성

git merge 이름 — 브랜치 합치기

📸 이 슬라이드를 스크린샷 찍어두세요!
