⚙️

4 / 30
개발 구조
프론트엔드
Frontend
사용자가 직접 보고 만지는 화면 부분

담당하는 것

• 버튼, 메뉴, 레이아웃

• 색상, 글꼴, 애니메이션

• 사용자 입력 처리

• 반응형 디자인 (모바일 대응)

사용하는 언어

HTML CSS JavaScript React Vue

💡 1강에서 만든 홈페이지가 바로 프론트엔드입니다!

5 / 30
개발 구조
백엔드
Backend
사용자에게 보이지 않는 서버 쪽 처리. 데이터 저장, 로직 실행, 보안 담당.

담당하는 것

• 회원가입/로그인 처리

• 결제 처리

• 데이터 저장/불러오기

• 보안, 권한 관리

사용하는 언어

Python Node.js Java Go PHP

💡 비유: 식당의 주방 — 손님은 안 보이지만, 음식(데이터)을 만들어 보내줌

개발 구조

Database
데이터를 체계적으로 저장하고 관리하는 프로그램

📊 비유: 엑셀 시트 (초고성능 버전)

📋

회원 정보

이름, 이메일, 비밀번호

🛒

주문 내역

상품, 수량, 날짜

📝

게시글

제목, 내용, 작성자

대표적인 DB: MySQL, PostgreSQL, MongoDB, Firebase

개발 도구
Git & GitHub
Version Control & Code Hosting
코드의 변경 이력을 기록하고, 팀이 함께 작업하는 도구

💾 비유: 문서의 "변경 추적" 기능

Git

내 컴퓨터에서 변경 기록

"ctrl+Z의 무한 버전"

GitHub

인터넷에 코드 저장/공유

"코드용 Google Drive"

⚠️ Git ≠ GitHub. Git은 도구, GitHub는 서비스입니다

개발 방식

16 / 30
개발 방식
워터폴 (폭포수)
Waterfall
단계별로 한 방향으로 진행하는 전통적 개발 방식
📋 기획
→
🎨 디자인
→
💻 개발
→
🧪 테스트
→
🚀 출시

✅ 장점

계획이 명확, 진행 상황 파악 쉬움

❌ 단점

중간에 변경 어려움, 느림

17 / 30
개발 방식
애자일
Agile
작은 단위로 빠르게 만들고, 피드백 받고, 개선하는 반복 방식
🔄 스프린트 1
기획→개발→테스트
→
🔄 스프린트 2
피드백→수정→추가
→
🔄 스프린트 3
개선→완성→출시

💡 여러분이 Claude와 "만들어줘 → 수정해줘 → 다시 수정해줘" 한 것이 바로 애자일 방식!

18 / 30
개발 방식
MVP (최소 기능 제품)
Minimum Viable Product
핵심 기능만 넣은 가장 작은 버전을 먼저 만들어 시장 반응을 보는 것
🛴

MVP

일단 이동 가능!

🚲

개선 버전

더 편하게

🚗

완성 버전

풀 기능!

💡 처음부터 완벽하게 만들지 마세요. 핵심만 빠르게 → 피드백 → 개선!

19 / 30
개발 방식
배포
Deploy / Deployment
만든 소프트웨어를 실제 사용자가 쓸 수 있게 올리는 것
💻 내 컴퓨터
(개발 환경)
🚀
☁️ 서버
(배포 = 세상에 공개)
→
👥 사용자
(접속해서 사용)

배포 서비스: Vercel, Netlify, GitHub Pages (모두 무료 가능!)

20 / 30
개발 방식
CI/CD
Continuous Integration / Continuous Deployment
코드를 수정하면 자동으로 테스트하고 배포하는 시스템
✍️ 코드 수정
→
🤖 자동 테스트
→
✅ 통과하면
→
🚀 자동 배포

💡 비유: 공장의 컨베이어 벨트 — 부품 올리면 자동으로 조립·검사·출하
