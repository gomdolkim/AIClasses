설치

ECC 설치 방법

핵심 슬래시 커맨드

가장 많이 쓰는 ECC 스킬들 📋

10 / 30
스킬

/plan — 구현 계획 세우기

코드를 쓰기 전에 단계별 계획을 세우는 스킬

• 요구사항 분석

• 리스크 평가

• 단계별 실행 계획

• 사용자 확인 후 실행

사용 예시

/plan 레스토랑 예약 웹앱을 만들어줘. 날짜 선택, 인원수, 메뉴 선택 기능이 필요해.

💡 코드 먼저 쓰지 말고, 항상 /plan 부터! 잘못된 방향으로 가는 것을 방지합니다

11 / 30
스킬

/tdd — 테스트 주도 개발

테스트를 먼저 작성하고, 그 다음 코드를 구현하는 방식

🔴 RED
테스트 작성
(실패)
→
🟢 GREEN
코드 구현
(통과!)
→
🔵 REFACTOR
코드 정리
(개선)
사용 예시

/tdd 장바구니에 상품 추가/삭제/수량 변경 기능을 구현해줘

12 / 30
스킬

/code-review — 코드 리뷰

작성한 코드의 품질, 보안, 성능을 자동으로 검사

• 버그 가능성 탐지

• 보안 취약점 확인

• 코드 스타일 일관성

• 성능 개선 제안

사용 예시

/code-review

→ 현재 변경된 코드를 자동으로 분석해줍니다

💡 코드를 작성한 직후에 실행하면 실수를 바로 잡을 수 있어요!

19 / 30
실전

Step 1: 기획 (/plan)

Claude Code에서 입력

/plan 개인 포트폴리오 웹사이트를 만들어줘. 자기소개, 프로젝트 갤러리, 연락처 폼이 필요해. Next.js + Tailwind CSS로 만들어줘.

Claude가 해주는 것:

✅ 요구사항 분석 및 정리

✅ 파일 구조 설계

✅ 단계별 실행 계획

✅ "이대로 진행할까요?" 확인 후 시작

20 / 30
실전

Step 2: 개발 (/tdd)

계획 승인 후 자동으로

TDD 방식으로 각 컴포넌트를 순차적으로 구현합니다

Claude가 자동으로:

🔴 테스트 먼저 작성

🟢 테스트를 통과하는 코드 구현

🔵 코드 정리 및 최적화

📂 파일 생성, 패키지 설치 — 모두 자동

21 / 30
실전

Step 3: 검증 (/verify)

개발 완료 후

/verify

한 번의 명령으로 전체 검증:

✅ 빌드 성공 여부

✅ 모든 테스트 통과

✅ 코드 스타일 검사 (린팅)

✅ 보안 취약점 스캔

💡 문제가 발견되면 자동으로 수정 제안까지 해줍니다!

22 / 30
실전

Step 4: 배포 (Git + Vercel)

# 검증 완료 후, 5강에서 배운 방법으로 배포!

$ git add .
$ git commit -m "포트폴리오 사이트 완성"
$ git push

# → Vercel이 자동으로 배포! 🚀

🎉 /plan → /tdd → /verify → git push — 이것이 ECC 실전 워크플로우!

25 / 30
팁

ECC 효율적으로 쓰는 5가지 팁

1. 항상 /plan 먼저 — 계획 없이 코딩하지 마세요

2. 프롬프트는 구체적으로 — "웹사이트" 보다 "React + Tailwind 포트폴리오"

3. /docs 활용 — 라이브러리 사용법이 헷갈리면 문서부터 검색

4. 작게 나눠서 요청 — 한 번에 전체보다, 기능별로 하나씩

5. 결과가 이상하면 /code-review로 바로 확인! 빠른 피드백이 핵심

실습 타임!

실습: ECC로 웹사이트 만들기

Step 1: 터미널에서 Claude Code 실행

$ claude

Step 2: 계획 세우기

/plan 내 이름, 취미, 좋아하는 것을 소개하는 개인 홈페이지를 만들어줘. HTML + CSS 하나의 파일로, 예쁘게 디자인해줘.

Step 3: 계획 확인 후

네, 진행해주세요!

⏱️ 10분간 자유롭게 만들어보세요!
